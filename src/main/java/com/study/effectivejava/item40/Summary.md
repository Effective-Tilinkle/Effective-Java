# @Override 애너테이션을 일관되게 사용하라
- 사용하지 않는 경우 인간적인 실수로 인해, 메서드 오버로딩이 될 수도 있음

- 예외적으로 사용가능한 경우는 1가지
  - 구체 클래스에서 상위 클래스의 추상메서드를 재정의한 경우엔 안달아도 된다. 어차피 구현하지 않은 추상메서드는 컴파일러가 잡아주기 때문
